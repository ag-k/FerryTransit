# タスク完了時のチェックリスト

## 必須確認事項
コード修正・機能追加後は以下を必ず実行：

### 1. ビルド確認 ✅
```bash
npm run build
```
- エラーなくビルドが完了することを確認
- ビルドエラーがある場合は必ず修正

### 2. ユニットテスト実行 ✅
```bash
npm run test:unit
```
- 関連するテストがすべてパスすることを確認
- テストが失敗した場合は、コードまたはテストを修正
- 新機能の場合は、対応するテストを新規作成

### 3. 型チェック実行 ✅
```bash
npm run typecheck
```
- TypeScriptの型エラーがないことを確認
- any型を使用していないことを確認

### 4. リントチェック実行 ✅
```bash
npm run lint
```
- ESLintエラー・警告を修正

## 追加確認事項

### 国際化対応
- 新しいテキストはすべて i18n キーとして追加
- ja.json と en.json の両方に翻訳を追加

### アクセシビリティ
- 新しいUIコンポーネントにARIA属性を適切に設定
- キーボードナビゲーションが機能することを確認

### レスポンシブデザイン
- モバイル・タブレット・デスクトップで表示確認
- Tailwindのレスポンシブクラスを使用

### Firebase ルール
- 新しいコレクション追加時はルールも更新
- 必要に応じて `firebase deploy --only firestore:rules` を実行

## コミット前の最終確認
1. 不要なconsole.logが残っていないか
2. コメントアウトされたコードが残っていないか
3. 機密情報がハードコードされていないか
4. 関連ドキュメントの更新が必要か

## トラブルシューティング
- ビルドエラー: エラーメッセージを確認し、該当箇所を修正
- テスト失敗: テストの期待値と実装が一致しているか確認
- 型エラー: 適切な型定義を追加または修正
- リントエラー: ESLint設定に従ってコードを修正
